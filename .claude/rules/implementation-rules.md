# 구현 코드 작성 규칙

코드 구현 시 코드 품질을 위해 다음을 반드시 준수할 것.

## 규칙

- Controller는 `@RestController`를 이용하여 구현한다.
- 모든 Service 클래스는 상위 계층과 데이터를 주고받을 때 인자는 원시값을 받고 리턴값은 DTO를 이용하도록 구현한다.
- Entity -> DTO 변환 책임은 DTO가 `from` 메서드를 통해 갖도록 구현한다.
- VO가 아닌 모든 도메인 엔티티는 `OffsetDateTime` 타입으로 생성일자와 수정일자를 관리하도록 구현한다.
- 수정 일자는 첫 엔티티 생성 시점에는 null로 초기화하고, 이후 수정 시점에만 값을 갱신하도록 구현한다.
- 한 도메인 구현 시 필요한 다른 도메인은 구현하지 않고 `PROGRESS.md` 파일에 구현이 필요한 도메인과 구현 순서를 기록한다.
