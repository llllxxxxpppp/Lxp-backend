# 강좌 컨텐츠 관리 바운디드 컨텍스트

## 강좌 서비스 비즈니스 규칙

- 비공개 처리를 위한 요청을 받으면 강사와 어드민인지 확인 한다.
- 강좌 생성 요청을 처리한다.
- 강좌 수정 요청을 처리한다.
- 강좌 비공개 처리한다.
- 강사의 강좌 업로드 요청을 처리한다.
- 공개 요청을 강사가 한건지 확인한다.
- 구독자(유효한 구독권을 가진 회원)는 강좌를 신고할 수 있다.

## 공통 VO

### Title

- 강좌, 강의, 미션의 제목을 표현한다. (String 타입)
- 비어있을 수 없다.
- 100자 초과 불가.

### Status

- 공개(PUBLIC) / 비공개(PRIVATE) 상태를 표현한다. (열거형)

### InstructorId

- 강사(회원)를 Id를 표현한다. (Long 타입)

### CourseId

- 강좌 Id를 표현한다. (Long 타입)

### LectureId

- 강의 Id를 표현한다. (Long 타입)

### MissionId

- 미션 Id를 표현한다. (Long 타입)

## 모든 애그리거트 공통 규칙

- 강좌, 강의, 미션 공통
  - 생성은 강사만 할 수 있다.
  - 공개는 강사만 할 수 있다.
  - 비공개는 강사와 어드민만 할 수 있다.
  - 삭제는 강사와 어드민만 할 수 있다.
- 공개 상태에서는 강사, 강의, 강좌의 정보를 수정할 수 없다.
- 수정 가능한 정보가 수정되면 수정 일시를 업데이트 한다.
- 삭제된 강좌, 강의, 미션은 어떠한 데이터도 수정할 수 없다.

- 불변 데이터 필드
    - 객체 Id: Long 타입
    - 생성 일시

- 가변 데이터 필드
    - soft delete 플래그: bool 타입
    - 생성자 ID: InstructorId VO
    - 제목: Title VO
    - 설명: 텍스트, 4096자 이내, NOT NULL
    - 수정 및 삭제 일시: 객체 생성 시 NULL

- 인터페이스
    - Sortable: 강의와 미션이 구현하여 강좌 단위로 순서를 변경할 수 있도록 한다.

## 강좌 애그리거트 도메인 모델 규칙 (root)

- 강좌는 비공개 상태로 생성된다.
- 강좌는 강의와 미션을 1개 이상 포함해야 공개할 수 있다.

- 강좌는 Sortable 객체 리스트를 가진다.
- Sortable 객체들의 노출 순서를 변경할 수 있다.
- Sortable 객체들은 강좌 단위로 고유한 순번을 가진다.

- 가변 데이터 필드
    - 썸네일 URI: 상대 주소 URI, Nullable

## 강의 애그리거트 도메인 모델 규칙

- 강의는 공개 상태로 생성된다.
- 강의는 Sortable 객체를 구현한다.

- 가변 데이터 필드
    - 컨텐츠 URI: 상대 주소 URI, NOT NULL
    - 자료 타입: String, 확장자, NOT NULL
    - 썸네일 URI: 상대 주소 URI, Nullable

## 미션 애그리거트 도메인 모델 규칙

- 미션은 공개 상태로 생성된다.
- 미션은 Sortable 객체를 구현한다.

- 가변 데이터 필드
    - 문제: String, 4000자 이내, NOT NULL
    - 모범 답안: String, 4000자 이내, NOT NULL
