# 회원 관리 바운디드 컨텍스트

## 도메인 모델 규칙

### 공통

- 회원 유형은 일반 회원, 강사, 어드민으로 나누어진다. JPA SINGLE_TABLE 상속으로 구현할 것.
- 회원 엔티티의 이메일을 username으로 사용한다.
- 회원 가입 시 이메일은 비어있을 수 없다.
- 이메일은 유효한 이메일 형식이어야 한다.
- 회원 가입 시 패스워드는 비어있을 수 없다.
- 회원 스스로 패스워드를 수정할 수 있다.
- 회원 생성 일시를 저장한다.
- 회원 정보 수정 일시를 저장하고, 수정 시 업데이트 한다.
- 회원 정지 일시를 저장한다. NULL이면 정지하지 않은 상태이다.
- 회원 탈퇴는 soft delete로 처리한다.
- 회원 생성 일시를 제외한 나머지 일시들은 처음에 NULL 값으로 채운다.
- 회원 ID는 MemberId 라는 record VO 객체를 만들어 사용한다.

### 일반 회원

- 회원 정보는 회원 본인이 수정할 수 있다.
- 회원은 탈퇴할 수 있다.
- 회원 탈퇴 일시를 저장한다. NULL이면 탈퇴하지 않은 상태이다.

### 강사

- 강사는 프로필을 가진다.
- 강사 프로필은 이름, 자기소개, 프로필 사진이 들어간다.
- 강사 프로필은 강사 본인이 수정할 수 있다.
- 강사 정지는 soft delete로 처리한다.

### 어드민

## 서비스 비즈니스 규칙

### 공통

- 회원의 이메일은 중복될 수 없다.
- 탈퇴된 회원은 이메일 앞 3글자만 보여준다.

### 일반 회원

- 회원은 강사, 회원을 신고할 수 있다.
- 회원 신고 생성 전 회원이 유효한 구독권을 소유중인지 검사한다.
- 회원 정지 -> 회원 정지 이벤트 발행 -> 회원의 활성화된 구독권을 정지 처리한다.

### 강사

- 강사는 어드민에 의해 정지될 수 있다.
- 강사 정지 -> 정지 이벤트 발행 -> 해당 강사가 작성한 강좌들을 비공개 처리한다.

### 어드민

- 강사 유형의 회원은 어드민만 생성 가능하다.
- 어드민만 강사를 정지할 수 있다.
